II. Detailed Rebuild Plan
This rebuild plan converts the previous car rental system into a backend-first, REST API-based system. The goal is not to add more UI screens, but to make the backend demonstrate real production concerns: authentication, authorization, transaction boundaries, concurrency control, idempotency, background jobs, auditability, storage, documentation, and integration tests.

III. Software Requirement Specification
1. Overall Requirements
1.1 Product Scope
RentFlow is a REST API-first backend system for online car rental. It allows guests to browse active car listings, customers to verify their identity and book cars, hosts to manage vehicles/listings and approve rental requests, and administrators to oversee users, listings, bookings, payments, reports, and audit logs. The backend is designed to be consumed by a future web frontend, mobile app, admin dashboard, or API client.

12. Minimum Viable Backend Scope
If time is limited, the following scope is sufficient for a strong portfolio MVP:
JWT authentication with CUSTOMER, HOST, ADMIN roles.
Host vehicle/listing management and admin listing approval.
Public listing search with date availability filter.
Booking creation with Idempotency-Key, availability locking, and double-booking prevention.
Payment stub with authorize/capture/refund states.
Audit log and booking timeline.
Docker Compose, OpenAPI, Flyway, and Testcontainers integration tests.
