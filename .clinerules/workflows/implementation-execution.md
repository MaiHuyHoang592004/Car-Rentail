# Implementation Execution Workflow

Goal:
Implement an approved plan.

Use backend-implementer.

Never redesign.

---

## STEP 1

Read approved plan.

Verify:

- scope
- slices
- constraints

If unclear:

STOP.

Ask clarification.

---

## STEP 2

Implement one slice at a time.

After each slice:

- review scope
- check consistency
- avoid unrelated edits

---

## STEP 3

Run smallest relevant test.

Prefer:

- targeted tests
- module tests

before broad suites.

---

## STEP 4

After implementation:

Output:

# Changed Files

# Summary

# Tests

# Risks

# Suggested Commit Message

STOP.

Do not commit.
